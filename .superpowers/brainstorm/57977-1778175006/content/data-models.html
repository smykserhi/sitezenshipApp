Data Models

Two MongoDB collections — User and Progress

👤 User Model

users collection
{
  _id:       ObjectId,
  email:     String (unique, required),
  password:  String (bcrypt hashed),
  name:      String,
  createdAt: Date
}

📊 Progress Model

progress collection
{
  _id:    ObjectId,
  userId: ObjectId (ref: User),

  education: {
    lastQuestionIndex: Number,   // where user left off (0–99)
    completedSections: [String]  // e.g. ["A","B","C"]
  },

  practice: {
    sessions: [{
      date:       Date,
      score:      Number,        // correct answers
      total:      Number,        // always 20
      passed:     Boolean,       // score >= 12
      questionIds:[Number]       // which 20 questions were asked
    }]
  },

  audio: {
    sessions: [{
      date:    Date,
      score:   Number,
      total:   Number,
      passed:  Boolean
    }]
  }
}

📚 Questions Data (not in DB)

All 100 questions are stored in data/questions.js — a static JS file parsed from Questions.md. Each question has:

{ id, section, question, answers: [String], fakeAnswers: [String] }

fakeAnswers = 5 plausible-but-wrong answers generated per question, stored statically. In Practice Mode, 1 correct + 5 fake = 6 choices shown to user.
